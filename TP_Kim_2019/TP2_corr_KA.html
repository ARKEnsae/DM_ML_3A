On remarque remarque qu'en proportition, un vin rouge ($y = 1$) est plus souvent prédit comme vin blanc ($\hat y = 0$) que l'inverse.

On remarque en consultant la matrice de confusion que l’échantillon de test contient nettement plus de vins blancs que de vins rouges. Ce n'est pas grave si la proportion de vin blanc est la même dans le test et dans le témoin et on n'a pas besoin d'avoir moins d'erreur pour prédire un vin rouge qu'un vin blanc. Par contre si on veut détecter tout le temps les vins rouges, ça peut être problématique. => On en déduit que lorsqu’on évalue la qualité des prédicteurs avec l’échantillon de test, la justesse des prédicteurs sur les vins blancs influence davantage le score que la justesse sur les vins rouges.

from sklearn.utils import resample, shuffle

def balanced_data(X, y):
    # 0 : white
    # 1 : red
    label_counts = y.value_counts()
    X_white = X[y==0]
    y_white = y[y==0]
    X_less_white, y_less_white = resample(X_white,
                                          y_white,
                                          n_samples=label_counts[1],
                                          replace=False)
    X_ = pd.concat([X_less_white, X[y==1]])
    y_ = pd.concat([y_less_white, y[y==1]])
    X_, y_ = shuffle(X_,y_)
    return(X_, y_)

X_test_, y_test_ = balanced_data(X_test, y_test)

Le score est plus faible sur le nouveau jeu de données test_ (contenant la même proportion de vins blancs et de vins rouge) que sur le jeu test originel, ou de manière équivalent l'erreur est plus importante. Les vins blancs sont beaucoup plus nombreux que les vins rouges dans le jeu de données d'entrainement, et l'algorithme, qui est entrainé pour minimiser l'erreur empirique de ce jeu de données, sera moins performant.

De manière générale, pour que l'algorithme fonctionne correctement, les distributions des données d'entrainement et de test doivent être semblables.

=> Il faut autant de vin rouge que de vin blanc dans le jeu de test et d'entraînement. On augmente artificiellement le nombre de vin rouge.

def shift_image(image,direction):
    dx, dy = direction
    x_shifted_image = np.zeros((8,8))
    if dx > 0:
        x_shifted_image[:,dx:] = image[:,:-dx]
    elif dx < 0:
        x_shifted_image[:,:dx] = image[:,-dx:]
    else:
        x_shifted_image = image 
    xy_shifted_image = np.zeros((8,8))
    if dy > 0:
        xy_shifted_image[:-dy] = x_shifted_image[dy:]
    elif dy < 0:
        xy_shifted_image[-dy:] = x_shifted_image[:dy]
    else:
        xy_shifted_image = x_shifted_image
    return xy_shifted_image
